Episode Runtime Shaping Prototype

Review where a long-form episode spends its time, compare the current runtime against a target, and hand tightening decisions back to the surfaces that already own pauses, segments, and intro or outro edits.

Runtime Overview

68m Current finished runtime, including intro and outro.

+8m Difference from target, shown as guidance rather than a hard cap.
over target The episode runs longer than the target, so the creator sees grouped opportunities to tighten without forcing any cut.

Runtime Decisions

Runtime shaping stays overview-first. Each tightening action opens the surface that already owns that edit, rather than turning this step into a second timeline editor.

Pause and dead-air trims Open pause and cross-talk cleanup when silence or repeated restarts are the main runtime problem.

Segment length drift Open show segment review when one section is much longer than the template or expected show shape.
segment review template-aware
Optional intro or outro Open intro and outro builder when the episode can shorten the opening or closing without changing the conversation.
